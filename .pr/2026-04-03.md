# Ship v3.0.0 ‚Äî C backend, bilingual syntax, the snake bites its own tail

The compiler used to emit LLVM IR that required SSA renaming, PHI nodes, and block terminators ‚Äî machinery that generated 2,921 errors when the self-hosted compiler tried to compile itself. This PR replaces all of that with a C backend where `break` is just `break` and `if/else` is just `if/else`. It also gives the language its own voice: bilingual keywords (Spanglish and English), indentation-based syntax, and a unified type system under `tipo` and `modo`.

The self-hosted compiler (11,363 lines of Mapanare) now emits 48,356 lines of C that gcc compiles and links into a 2MB binary with zero errors. The `mapanare run` command defaults to C (via gcc) ‚Äî no llvmlite dependency needed. A migration tool converts existing v2 code, and 99 `.mn` files across the stdlib, examples, and ecosystem packages have already been migrated. The old brace syntax still compiles ‚Äî both forms are accepted.

### Highlights

- New C emit backend: `mapanare emit-c file.mn` compiles any Mapanare program to portable C99 that links against the existing C runtime
- `mapanare run` now uses the C backend by default ‚Äî no LLVM or llvmlite needed
- Bilingual keywords: write `pon x = 42` or `let x = 42`, `si cond` or `if cond`, `da val` or `return val` ‚Äî both compile identically
- Indentation syntax: `fn main:` with colon+indent works alongside traditional braces
- `tipo` unifies struct and enum: `tipo Point` for structs, `tipo Color` with `| Red | Green` for enums
- `modo` replaces `trait`, `@Name { ... }` replaces `agent Name { ... }`, `->` / `<-` for channel declarations
- `mapanare migrate --to=v3` converts v2 code to v3 keywords automatically
- `pip install mapanare` works without llvmlite (moved to `[llvm]` extras)

<summary>Technical changes</summary>

**C emit backend (`mapanare/emit_c.py`, 2,239 lines)**
- MIR-to-C emitter covering all 45 MIR instruction types: const, copy, binop, unaryop, call, return, jump, branch, switch, struct_init, field_get/set, list_init/push, index_get/set, map_init, enum_init/tag/payload, wrap_some/none/ok/err, unwrap, closure_create/call, env_load, interp_concat, assert, agent_spawn/send/sync, signal_init/get/set/computed/subscribe, stream_init/op
- PHI elimination: phi destinations declared at function scope, predecessor blocks assign values before terminators
- Topological sort for struct/enum/Option/Result type emission order with cycle detection
- Circular Option types (TypeInfo, Scope) use heap-allocated pointer values with dereference on extraction
- Forward-referenced enum/struct fields in cyclic types emit as `void*` with `memcpy`-based access
- Type propagation pass infers C types from enum constructors, function return types, Copy sources, WrapSome values, and field access patterns
- SSA variable reuse guards: `memcpy` for Const‚Üístruct, Call‚Üístruct, BinOp struct operand casting
- Argument coercion via `memcpy` when function parameter types differ from argument types
- Namespace resolution: `Program_start` ‚Üí `start` for struct-namespaced constructor calls
- `main()` wrapper auto-generated for `compile_and_print` entry points (self-hosted compiler)

**Bilingual grammar (`mapanare/mapanare.lark`)**
- 17 keyword pairs using regex terminals with word-boundary lookahead (`(?![a-zA-Z0-9_])`) and priority `.2`: `pon`/`let`, `da`/`return`, `si`/`if`, `sino`/`else`, `cada`/`for`, `en`/`in`, `mien`/`while`, `sal`/`break`, `sigue`/`continue`, `nada`/`none`, `yo`/`self`, `usa`/`import`, `tipo`/`type`, `modo`/`trait`/`way`
- `tipo_def` rule: dispatches to `StructDef` (plain fields) or `EnumDef` (`|`-prefixed variants) based on body content
- `at_agent_def` rule: `@Name { ... }` with `agent_channel_in` (`->`) and `agent_channel_out` (`<-`)
- `continue_stmt` rule with `KW_CONTINUE` terminal
- `BAR` terminal for tipo variant syntax

**Indentation preprocessor (`mapanare/parser.py`)**
- Source-level converter: lines ending with `:` ‚Üí `{`, dedent ‚Üí `}`
- Continuation handling: `sino:`/`else:` merges `}` with opening `{` on one line
- Doc comments (`///`) trigger block-close on dedent
- `fn name:` with no parens ‚Üí `fn name() {` (zero-arg function)
- Falls back to braces when no colon blocks detected ‚Äî full backward compatibility
- `tipo_def`, `at_agent_def`, `modo_def`, `continue_stmt` transformer handlers
- `ContinueStmt` AST node with semantic checker and MIR lowerer support (jump to loop header)

**CLI changes (`mapanare/cli.py`)**
- `emit-c` subcommand with `_compile_to_c()` pipeline
- `migrate` subcommand wired to `mapanare/migrate.py`
- `cmd_run` defaults to C backend (`_run_c_source()`: emit C ‚Üí gcc ‚Üí tmpdir ‚Üí run)
- `--release` flag on `run` for LLVM JIT backend
- `_run_c_source()` helper: compiles against `mapanare_core.c`, links with `-lm -lpthread`

**Migration tool (`mapanare/migrate.py`, 373 lines)**
- Keyword replacement: `let`‚Üí`pon`, `return`‚Üí`da`, `if`‚Üí`si`, `else`‚Üí`sino`, `for`‚Üí`cada`, etc.
- Structural transforms: `struct`‚Üí`tipo`, `enum`‚Üí`tipo` with `|` variant prefix, `trait`‚Üí`modo`, `agent`‚Üí`@Name`
- Comma-separated enum variants detected and prefixed with `|` inside `tipo` blocks
- `--dry` preview, `--check` CI gate, `--style=spanglish|english` modes
- Braces preserved (robust ‚Äî brace-to-indent conversion deferred)

**Self-hosted C emitter (`mapanare/self/emit_c.mn`, 770 lines)**
- Mirrors `emit_c.py` in Mapanare: preamble, struct/enum defs, string constants, function bodies, instruction dispatch
- 32 functions, 1 struct ‚Äî parses clean through the Python bootstrap compiler

**MIR optimizer (`mapanare/mir_opt.py`)**
- DCE preserves `compile_and_print`, `compile`, `version` as entry points
- Namespace resolution in call graph: `Program_start` ‚Üí `start` prevents false dead-code elimination

**Package configuration (`pyproject.toml`)**
- `llvmlite` moved from `dependencies` to `[llvm]` optional extras
- `dev` extras still include `llvmlite` for development

**Self-hosted compiler fixes (`mapanare/self/*.mn`)**
- Renamed `si` ‚Üí `si_idx`, `en` ‚Üí `en_idx`/`en_val`, `da` ‚Üí `da_val` across 5 modules to avoid bilingual keyword conflicts

**Migrated files (99 `.mn` files)**
- `tests/golden/` (15), `examples/` (12), `stdlib/` (39), `crawl/` (8), `dato/` (9), `scan/` (6), `fuzz/` (3), `tests/test_runner/fixtures/` (3)
- Keywords converted to Spanglish; braces and structure preserved
