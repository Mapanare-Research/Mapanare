# Ship v3.0.1: self-hosted compiler runs, four new syntax features

The snake finally bites its own tail. The self-hosted Mapanare compiler — 11,363 lines of `.mn` compiled through the C backend — now produces 77,000 lines of valid LLVM IR when fed its own source. It went from segfaulting on launch to self-compiling in a single release cycle, which required untangling a chain of type inference gaps, pointer tagging collisions, void* boxing mismatches, and memcpy overflow patterns that had been silently corrupting every struct larger than 16 bytes.

On the language side, v3.0.1 delivers four syntax features that complete the v3 design: `di` as a print keyword, `+` as a pub prefix, `...` for empty blocks, and implicit return. These are all additive — existing code keeps working. The `si`-as-match unification is deferred to v3.2 because LALR can't disambiguate `si expr { stmts }` from `si expr { arms }` without the indent preprocessor.

### Highlights

- Write `di "hello"` instead of `print("hello")` — the `di` keyword is a first-class statement, and `print()` still works as a function call
- Mark definitions public with `+fn`, `+tipo`, `+struct` instead of `pub fn`, `pub tipo`, `pub struct`
- Use `...` as an empty block body: `fn todo() { ... }` is Mapanare's `pass`
- Last expression in a typed function is returned automatically: `fn double(x: Int) -> Int { x * 2 }`
- The self-hosted compiler binary (mnc-stage1) runs and self-compiles through the C backend
- 11 of 15 golden programs produce valid LLVM IR through mnc-stage1

<summary>Technical changes</summary>

**C emit backend (`mapanare/emit_c.py` — new, 2,372 lines)**
- String constants use `__attribute__((aligned(2)))` char arrays to prevent `.rodata` literals from landing at odd addresses, which collided with the runtime's LSB pointer tagging (`mn_tag_heap`/`mn_untag`)
- void*-boxed struct fields (forward-reference cycles) now heap-allocate on store and pointer-dereference on load, fixing the 288-byte-from-8-byte truncation in `DefResult.def_node`
- Per-variant tracking of boxed enum payload fields via `_boxed_fields` set with `(enum_name, variant, field_idx)` tuples
- `_propagate_types()` post-pass back-propagates `MnOption` types from struct field definitions, function parameter types, and enum payload types — resolves `MnOption_int64_t` → `MnOption_TypeExpr` mismatches
- All `memcpy(&dest, &src, sizeof(dest))` patterns replaced with `sizeof(src)` or `sizeof(field)` variants, with `memset(&dest, 0, sizeof(dest))` guard when destination is larger
- `StructInit` uses direct assignment when field types match instead of unconditional `memcpy`
- Return handler detects scalar-to-struct mismatch (unreachable match defaults) and zero-initializes instead of overflowing
- PHI store handler zero-initializes + copies `sizeof(src)` for cross-type merges

**MIR type inference (`mapanare/lower.py` — +186 lines)**
- `SomeExpr`/`OkExpr`/`ErrExpr` propagate inner type from wrapped value into `TypeInfo.args`
- `_lower_call` resolves return types for `NamespaceAccessExpr` callees (e.g., `Program::start` → `STRUCT:Program`)
- `_lower_namespace_access` emits `EnumInit` for payload-less enum variants instead of `Call` with unknown return type
- `_infer_iterable_elem_type` returns `mir_int()` for `RANGE` iterables
- `_lower_let` patches bare `Option`/`Result` types from let-binding type annotations via `_patch_wrap_inst`
- `_lower_construct` calls `_patch_arg_types_from_params` for struct literal field values
- `_lower_list` includes element type in `dest.ty.type_info.args`
- List `.push()` reuses the source variable name for in-place mutation, preventing SSA aliasing bugs in for loops
- `_fn_param_types` populated for struct constructors and impl methods during `_register_declarations`

**Grammar and parser (`mapanare.lark`, `parser.py`)**
- `KW_DI` terminal: `/di(?![a-zA-Z0-9_])/` — only matches bare `di`, not `print`
- `di_stmt: KW_DI expr` rule added to `?stmt` alternatives
- `ELLIPSIS: "..."` terminal; `block` rule accepts `LBRACE ELLIPSIS RBRACE` as empty block
- All definition rules (`fn_def`, `struct_def`, `enum_def`, `tipo_def`, `trait_def`, `agent_def`, `pipe_def`) accept `(KW_PUB | PLUS)?` prefix
- `_has_pub_prefix()` helper scans raw children for `PLUS` token without adding it to `_KEEP` filter set (which would leak the `+` operator into arithmetic AST nodes)

**AST and semantic (`ast_nodes.py`, `semantic.py`)**
- `PrintStmt(expr: Expr)` dataclass added after `AssertStmt`
- Semantic checker infers `PrintStmt.expr` type

**Self-hosted compiler (`mapanare/self/*.mn`)**
- Loop iteration limits raised from 50 to 200 across `emit_llvm.mn`, `lower.mn`, `semantic.mn` (68 structs exceeded the old limit, truncating type definitions)
- `lower_if` PHI emission: skip terminated branches, guard against type kind mismatch (`i64` vs `%enum.Expr`), emit return terminator in merge block when both branches terminate
- `lower_match` PHI: include all non-terminated arms with `zeroinitializer` for void results; unreachable merge uses function return type via `get_current_fn_ret_type` helper
- `lower_match` switch default: add `zeroinitializer` PHI entry from entry block when default targets merge directly
- `scripts/fix_stage2_ir.py`: post-processor that fixes 5 structural issues in stage2 LLVM IR (misplaced merge blocks, empty blocks, dead PHIs) caused by COW state threading

**Lint and CI**
- `ruff check mapanare/ runtime/`: 0 errors (removed unused variables, fixed imports)
- `black --check`: 0 reformats
- `mypy --ignore-missing-imports`: 0 errors (added type annotations for `_boxed_fields`)
